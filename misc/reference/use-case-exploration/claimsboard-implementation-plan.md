# Claimsboard — Implementation Blueprint

A self-contained Nuxt 4 app living at `/home/frank/repos/looka/claimsboard/`. It ships its own UI + API + SQLite on one Nitro origin, so it does **not** fold into the existing single-image Looka Docker service. Everything below is concrete and ordered to the 6 roadmap phases.

---

## 1. Monorepo integration

**Placement:** new workspace package at `looka/claimsboard/` (sibling to `frontend/` and `backend/`). One-word name, self-contained.

**`pnpm-workspace.yaml`** — add the package:
```yaml
packages:
  - frontend
  - backend
  - claimsboard
```
Keep `onlyBuiltDependencies` in the root `package.json` (already lists `better-sqlite3`, `esbuild`) — that covers claimsboard's native addon too. No root-script changes are required; run it with `pnpm --filter claimsboard dev`. Optionally add convenience scripts later, but the existing root `dev` (`pnpm -r --parallel dev`) would then also boot Nuxt — acceptable, but for the demo prefer running claimsboard on its own to keep its output separate (Nuxt is on :3000, WebSpatial frontend on :5173, backend on :8787 — no actual port clash).

**Tooling — own vs shared:**
- **tsconfig:** own. Nuxt generates `.nuxt/tsconfig.json`; `claimsboard/tsconfig.json` just `{"extends": "./.nuxt/tsconfig.json"}`. Do **not** chain `tsconfig.base.json` (its `lib: ["ES2023"]` with no DOM fights Nuxt's generated config).
- **Lint:** shared root ESLint. Root `eslint.config.ts` currently sets `react: true`; `@antfu/eslint-config` auto-enables Vue when a `vue` dependency is present in the workspace, so Vue SFCs get linted with no config change. Add `claimsboard/.nuxt/**`, `claimsboard/.output/**`, and `claimsboard/server/db/migrations/**` to the root `ignores` array. Still ESLint-only, no Prettier — matches AGENTS.md.
- **No `justfile`** in the monorepo (that's a gridfin thing); use pnpm scripts inside `claimsboard/package.json` mirroring the design plan's Getting-Started block: `dev`, `build`, `db:generate`, `db:migrate`, `db:seed`, `auth:generate`, `lint`.

**Deploy (now):** leave the Looka Coolify service untouched — it builds one image that serves the React SPA + Hono API on port 8787. Claimsboard is a different stack and a different origin, so wiring it into that image is wrong. For the hackathon demo it runs locally. If a hosted URL is wanted later, add a **second** Coolify application from the same repo with its own `claimsboard/Dockerfile` (`nuxt build` → `node .output/server/index.mjs`) and a named volume for `data/*.sqlite` + `uploads/`. Mark this **deferred** (Phase 6+), and note it in a short `claimsboard/README.md`.

## 2. Full Drizzle schema — `server/db/schema.ts`

One file. App tables + the four Better Auth tables (singular names). Money as **integer cents** (no floats). Datetimes as `integer({ mode: 'timestamp' })`; date-only (`lossDate`) as text ISO. Text primary keys (UUID) so app rows and Better Auth's text ids join cleanly. SQLite has no enums — use Drizzle `text(..., { enum: [...] })`.

```ts
import { sql } from 'drizzle-orm'
import { index, integer, sqliteTable, text, unique } from 'drizzle-orm/sqlite-core'

// ---- Better Auth (generated by @better-auth/cli, keep names singular) ----
export const user = sqliteTable('user', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  email: text('email').notNull().unique(),
  emailVerified: integer('email_verified', { mode: 'boolean' }).notNull().default(false),
  image: text('image'),
  role: text('role', { enum: ['handler', 'manager'] }).notNull().default('handler'), // additionalField; if a seeded signUpEmail errors on NOT NULL, drop .notNull() (keep .default)
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})

export const session = sqliteTable('session', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  token: text('token').notNull().unique(),
  expiresAt: integer('expires_at', { mode: 'timestamp' }).notNull(),
  ipAddress: text('ip_address'),
  userAgent: text('user_agent'),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})

export const account = sqliteTable('account', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  accountId: text('account_id').notNull(),
  providerId: text('provider_id').notNull(),
  accessToken: text('access_token'),
  refreshToken: text('refresh_token'),
  idToken: text('id_token'),
  accessTokenExpiresAt: integer('access_token_expires_at', { mode: 'timestamp' }),
  refreshTokenExpiresAt: integer('refresh_token_expires_at', { mode: 'timestamp' }),
  scope: text('scope'),
  password: text('password'), // email+password hash lives here
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})

export const verification = sqliteTable('verification', {
  id: text('id').primaryKey(),
  identifier: text('identifier').notNull(),
  value: text('value').notNull(),
  expiresAt: integer('expires_at', { mode: 'timestamp' }).notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' }),
  updatedAt: integer('updated_at', { mode: 'timestamp' }),
})

// ---- App tables ----
export const claims = sqliteTable('claims', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  claimNo: text('claim_no').notNull(),
  policyNo: text('policy_no').notNull(),
  line: text('line', { enum: ['auto', 'property', 'liability', 'health', 'travel'] }).notNull(),
  status: text('status', {
    enum: ['new', 'in_review', 'awaiting_docs', 'approved', 'paid'],
  }).notNull().default('new'),
  subStatus: text('sub_status', { enum: ['denied', 'reopened'] }),
  customer: text('customer').notNull(),
  lossType: text('loss_type'),              // water, fire, theft, collision, ...
  lossDate: text('loss_date'),              // ISO date (date-only)
  reportedAt: integer('reported_at', { mode: 'timestamp' }).notNull(),
  location: text('location'),
  description: text('description'),
  reserveCents: integer('reserve_cents').notNull().default(0),
  paidCents: integer('paid_cents').notNull().default(0),
  deductibleCents: integer('deductible_cents').notNull().default(0),
  settlementCents: integer('settlement_cents').notNull().default(0),
  assigneeId: text('assignee_id').references(() => user.id, { onDelete: 'set null' }),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
}, t => [
  unique('claim_no_uq').on(t.claimNo),
  index('claims_status_idx').on(t.status),
  index('claims_assignee_idx').on(t.assigneeId),
]) // array-form extras (object form deprecated since drizzle-orm 0.36)

export const claimParties = sqliteTable('claim_parties', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  claimId: text('claim_id').notNull().references(() => claims.id, { onDelete: 'cascade' }),
  role: text('role', {
    enum: ['insured', 'claimant', 'adjuster', 'third_party', 'witness', 'repair_shop'],
  }).notNull(),
  name: text('name').notNull(),
  contact: text('contact'),
  org: text('org'),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
}, t => [index('parties_claim_idx').on(t.claimId)])

export const claimDocs = sqliteTable('claim_docs', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  claimId: text('claim_id').notNull().references(() => claims.id, { onDelete: 'cascade' }),
  type: text('type', {
    enum: ['fnol', 'photo', 'police_report', 'estimate', 'invoice', 'proof_of_ownership',
      'medical_bill', 'adjuster_report', 'correspondence', 'release', 'id_declaration',
      'witness_statement', 'other'],
  }).notNull().default('other'),
  filename: text('filename').notNull(),
  url: text('url').notNull(),                 // e.g. /uploads/<claimId>/<file>
  mimeType: text('mime_type'),
  sizeBytes: integer('size_bytes'),
  uploadedBy: text('uploaded_by').references(() => user.id, { onDelete: 'set null' }),
  uploadedAt: integer('uploaded_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
}, t => [index('docs_claim_idx').on(t.claimId)])

export const claimEvents = sqliteTable('claim_events', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  claimId: text('claim_id').notNull().references(() => claims.id, { onDelete: 'cascade' }),
  type: text('type', {
    enum: ['created', 'note', 'status_change', 'payment', 'document', 'party', 'assignment'],
  }).notNull(),
  actorId: text('actor_id').references(() => user.id, { onDelete: 'set null' }),
  body: text('body'),
  meta: text('meta', { mode: 'json' }).$type<Record<string, unknown>>(),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
}, t => [
  index('events_claim_idx').on(t.claimId),
  index('events_created_idx').on(t.createdAt),
])

export type Claim = typeof claims.$inferSelect
// … $inferSelect/$inferInsert exports per table
```

Sub-states (`denied`, `reopened`) are a nullable column, surfaced as **filters**, not board columns, exactly as the plan states.

## 3. Seed plan — `server/db/seed.ts`

Run order: migrate, then create users **through Better Auth**, then patch the manager role, then bulk-insert claims/parties/docs/events. Never raw-insert users (passwords must hash via the sign-up flow).

```ts
// 1. runMigrations()  (better-sqlite3 migrator, see §7)
// 2. Create users via the auth server API — NEVER db.insert(user) (that skips password hashing)
const USERS = [
  { name: 'Mara', email: 'mara@claimsboard.test', role: 'manager' },
  { name: 'Jana', email: 'jana@claimsboard.test', role: 'handler' },
  { name: 'Sami', email: 'sami@claimsboard.test', role: 'handler' },
]
for (const u of USERS) {
  await auth.api.signUpEmail({ body: { name: u.name, email: u.email, password: DEV_PASSWORD } })
  // additionalFields `defaultValue` is unreliable on the Drizzle adapter (better-auth #2674),
  // so set the role explicitly right after each sign-up (all three, not just Mara):
  await db.update(user).set({ role: u.role }).where(eq(user.email, u.email))
}
// 3. Bulk-insert claims / parties / docs / events (distribution below).
```

- `DEV_PASSWORD` shared, read from env with a fallback (`claimsboard-dev`).
- **~18 claims** spread deliberately so the board and dashboard look alive on first load:
  - **Status distribution:** ~3 `new`, 5 `in_review`, 3 `awaiting_docs`, 3 `approved`, 4 `paid`; two carry a `subStatus` (one `denied`, one `reopened`) to exercise filters.
  - **Lines:** mix of `property` (water/fire/theft), `auto` (collision/theft), `liability`, `health`, `travel`.
  - **Assignees:** distribute across Jana/Sami with Sami intentionally **overloaded** (so Team's "needs attention / reassign" list has something to show); a few unassigned `new` claims.
  - **Money:** realistic cents — reserve 200k–1.4M cents (€2k–€14k), some `paid` rows with `paidCents` set and `updatedAt` inside the last 7 days so "paid-this-week" is non-zero.
  - **Timestamps:** `reportedAt` staggered over the last ~30 days; make 2–3 open claims older than the SLA threshold (see §9) so "past-SLA" > 0.
- **Parties:** each claim gets 1–3 (`insured` always; `adjuster`/`claimant`/`witness`/`repair_shop` sprinkled).
- **Docs:** 5–8 claims get 2–3 rows each referencing **shipped placeholder assets** in `public/samples/` (`fnol.pdf`, `photo-1.jpg`, `estimate.pdf`, …) so the Documents tab and board thumbnails render without a real upload.
- **Events:** every claim gets a `created` event; status-advanced claims get matching `status_change` + `payment`/`document` events, timestamped to feed the dashboard's **recent-activity** feed. Use the exact demo strings from the wireframe ("Sami approved #4459 · €2.1k", etc.) so the demo matches the design.

Idempotency: wipe app tables + auth tables at the top (dev-only) before seeding, so re-running is clean.

## 5. Middleware & session

Better Auth client via `createAuthClient` from `better-auth/vue` in `app/lib/auth-client.ts`. To keep session reads **SSR-safe**, hydrate once in a plugin and read from shared state in middleware (avoids per-navigation refetch and header-forwarding pitfalls):

```
app/
├─ lib/auth-client.ts              # createAuthClient({ baseURL }) → signIn/signUp/signOut/useSession
├─ plugins/auth.ts                 # on server+client: authClient.getSession() → useState('cb-user')
├─ composables/useAuth.ts          # wraps useState: user, isAuthenticated, isManager, sign*/refresh
└─ middleware/
   ├─ auth.ts                      # named: redirect to /auth?next=... if no session
   └─ manager.ts                   # named: redirect to /dashboard if role !== 'manager'
```

- **`plugins/auth.ts`** runs universally; on the server it forwards incoming cookies to the session call — `authClient.getSession({ fetchOptions: { headers: useRequestHeaders(['cookie']) } })` — storing the result in `useState('cb-user')` so SSR and client agree (no hydration flash).
- **`useAuth()`** exposes `user`, `isAuthenticated`, `isManager = computed(() => user.value?.role === 'manager')`, plus `signIn/signUp/signOut` (which call the client then refresh the `useState`). This is the direct analogue of gridfin's `useAuth` singleton.
- **`middleware/auth.ts`** (applied via `definePageMeta({ middleware: 'auth' })` on every app page, or as a route-rule group) reads `useAuth().isAuthenticated`; if false, `return navigateTo({ path: '/auth', query: { next: to.fullPath } })`.
- **`middleware/manager.ts`** stacks on top for `team.vue` only: `if (!isManager.value) return navigateTo('/dashboard')`. Defence-in-depth: the Team **API** endpoints also re-check role server-side (§7) — middleware is UX, not security.

**Better Auth instance (`server/lib/auth.ts`)** — exact shape (adapter import path matters):

```ts
import { betterAuth } from 'better-auth'
import { drizzleAdapter } from 'better-auth/adapters/drizzle' // ships INSIDE better-auth — do NOT install @better-auth/drizzle-adapter
import { db } from '~~/server/db/client'
import * as schema from '~~/server/db/schema'

export const auth = betterAuth({
  database: drizzleAdapter(db, { provider: 'sqlite', schema }),
  emailAndPassword: { enabled: true },
  user: { additionalFields: { role: { type: 'string', input: false, required: false, defaultValue: 'handler' } } },
  // secret + baseURL read from BETTER_AUTH_SECRET / BETTER_AUTH_URL (see Appendix .env.example)
})
```

**Reusable CRUD pattern.** Every mutating handler follows the same three-step shape (use `safeParse` + an explicit 400 rather than letting a thrown `ZodError` bubble to a 500):

```ts
export default defineEventHandler(async (event) => {
  const me = await requireSession(event)                    // 401 if no session
  const body = await readValidatedBody(event, ClaimPatch.parse) // Zod → 400 on failure
  const id = getRouterParam(event, 'id')
  const [prev] = await db.select().from(claims).where(eq(claims.id, id))
  if (!prev) throw createError({ statusCode: 404, statusMessage: 'Claim not found' })

  const [row] = await db.update(claims)
    .set({ ...body, updatedAt: new Date() })
    .where(eq(claims.id, id)).returning()

  // auto-append events on meaningful transitions:
  if (body.status && body.status !== prev.status)
    await appendEvent(id, 'status_change', me.id, null, { from: prev.status, to: body.status })
  if (body.paidCents && body.paidCents !== prev.paidCents)
    await appendEvent(id, 'payment', me.id, null, { cents: body.paidCents - prev.paidCents })

  return row
})
```

- **`requireSession(event)`** = `auth.api.getSession({ headers: event.headers })` → throw `createError({ statusCode: 401 })` if null; returns the user. **`requireManager`** additionally checks `role === 'manager'` (guards `team.get.ts`; the middleware in §5 is only UX).
- **Error shape:** always `createError({ statusCode, statusMessage, data })` — Nitro serialises it to `{ statusCode, statusMessage, data }` which `useFetch`/`$fetch` surface as `error.value`. Client catches and toasts.
- **Validation:** one `server/utils/validators.ts` of Zod schemas (`ClaimCreate`, `ClaimPatch`, `PartyCreate`, `PartyPatch`, `DocCreate`, `EventCreate`) reused by handlers via `readValidatedBody` / `getValidatedQuery`.
- **Event auto-append** centralised in `appendEvent()`: called on create (`created`), status change (`status_change`), payment (`payment`), doc upload (`document`), party add (`party`), reassign (`assignment`). This is what keeps the dashboard feed and the claim Activity tab truthful with zero extra client work.
- **Server-only isolation:** `server/lib/auth.ts`, `server/db/*` import `better-sqlite3` (native, synchronous) and must **never** be imported from `app/`. Access data only through `/api/*`. The client uses `better-auth/vue`'s `authClient`, never `server/lib/auth.ts`.

## 8. Documents — mock upload

**Recommendation: write to a local `uploads/` dir on disk, serve via a Nitro route, store only metadata in `claim_docs`.** Rationale: real file semantics (thumbnails, downloads, sizes) with zero cloud infra; keeps the SQLite file small and fast; trivially resettable. Storing blobs in SQLite bloats the DB and complicates the demo for no gain.

- **Upload (`documents/index.post.ts`):** `readMultipartFormData(event)`, write each part to `./uploads/<claimId>/<uuid>-<safeName>` (create dir with `mkdirSync recursive`), insert a `claim_docs` row with `url = /uploads/<claimId>/<file>`, `mimeType`, `sizeBytes`, `uploadedBy = me.id`; `appendEvent(..., 'document', ...)`.
- **Serve (`server/routes/uploads/[...file].get.ts`):** resolve the path **safely** (reject `..`), `return sendStream(event, createReadStream(abs))` with the right `Content-Type`. Simpler than teaching Nitro publicAssets about a runtime dir, and keeps auth optional (fine for demo).
- **Seed docs** point at committed placeholders under `public/samples/*` (served by Nuxt's static handler), so the board/Documents tab look populated before anyone uploads.
- `uploads/` is gitignored (add `claimsboard/uploads/` to `.gitignore`), same discipline as `backend/data/`.

## 9. Dashboard aggregation

Compute in **SQL** in a single `server/api/dashboard.get.ts` — correct, and the natural OpenClaw-skill surface later. Dataset is tiny, but SQL groupBy is cleaner than pulling all rows and reducing in JS.

- **KPI cards:**
  - `open` = `count(*) where status != 'paid'`.
  - `awaiting_docs` = `count(*) where status = 'awaiting_docs'`.
  - `past_sla` = `count(*) where status not in ('paid','approved') and reportedAt < now - SLA_DAYS` (define `SLA_DAYS = 7` as a const in `server/utils/sla.ts`).
  - `paid_this_week` = `sum(paidCents) where status = 'paid' and updatedAt >= now - 7d` (return cents, format client-side).
- **By-status:** `select status, count(*) group by status` → map to the five ordered columns.
- **My queue:** scoped to the session user: `where assigneeId = me.id and status != 'paid' order by reportedAt limit 6`.
- **Recent activity:** `select … from claim_events order by createdAt desc limit 15`, joined to `claims` (claimNo/customer) and `user` (actor name) for display strings.
- Return one JSON object `{ kpis, byStatus, myQueue, activity }`; `dashboard.vue` does a single `useFetch`. **Team** endpoint reuses the same helpers, grouped by `assigneeId`.

## 12. Risks / gotchas / defer

- **Better Auth migrate ≠ Drizzle.** Only ever run `@better-auth/cli generate` (schema) → `drizzle-kit generate && migrate`. Never `better-auth migrate`. Keep table names **singular**.
- **Seed users only via `auth.api.signUpEmail`.** Raw `db.insert(user)` skips password hashing → logins silently fail. Sign-up defaults `role: 'handler'`; patch Mara → `manager` after.
- **Server-only leakage.** `better-sqlite3` is a native synchronous addon; if `server/lib/auth.ts` or `server/db/*` is imported into an `app/` component the client bundle breaks. Access data strictly through `/api/*`; client auth strictly through `better-auth/vue`.
- **Native build in pnpm.** `better-sqlite3` must be in root `onlyBuiltDependencies` (already is) or install won't compile the addon.
- **Drag lib + SSR.** Wrap the board in `<ClientOnly>`; `vue-draggable-plus` mutates the DOM and will otherwise cause hydration mismatches. Do optimistic move + rollback on PATCH failure.
- **SSR session flash.** Forward the `cookie` header in `plugins/auth.ts` on the server call, or the first paint shows logged-out chrome then flips. The `useState` hydration approach in §5 avoids it.
- **Nuxt UI v4 is Tailwind v4 / CSS-first.** No `tailwind.config`; theme lives in `@theme` + `app.config.ts`. Don't reach for a v3-style config.
- **Money precision.** Integer cents everywhere; format at the edge. Never store euros as float.
- **Lint.** Ensure antfu picks up Vue (add `vue` dep — it does automatically) and that `.nuxt/**`, `.output/**`, `server/db/migrations/**` are in root `ignores`.
- **Defer (out of scope now):** real authorization/ownership checks beyond role gating; file-type/size validation and virus concerns on upload; pagination on the claims table; websockets/live updates (dashboard is fetch-on-load); the Docker/Coolify second service; and the OpenClaw skill → agent → kami wrapping — the endpoints are shaped for it, but build only the app now.
