01

The reframe, not a rebuild

The industrial track asks for three hard things. Looka's spatial-panel architecture already answers the shape of all three — every agent is a floating window that reads a live feed and speaks what matters. We only change what each panel is pointed at.

What the brief demands

  • Spatial mapping of dynamic sites — factory floors, warehouses, things move.
  • Multi-system data fusion into one AR overlay — machines, blueprints, IoT, checklists.
  • Auto-documentation that writes itself — inspections, anomalies, repairs.
FUSE ▸

What Looka already is

  • Draggable WebSpatial panels placed around a room — spatial memory, at a glance.
  • A thin relay per agent → LLM + tools → one voice answer. Fusion is the default.
  • A voice orb that listens and logs hands-free — Frank's Realtime agent, shipped.

How it plugs into Looka

No new architecture. Each agent is the same per-member slice we already ship — a self-contained feature page and a thin relay — so four people build in parallel without collisions, and it stays the same PICO WebSpatial build that enters the primary track.

The same build, a second $600 pot

The primary entry is PICO Tech Layer — the spatial room itself. Pointing the panels at a factory reframes the exact same app as Industrial Operations: Iron Man HUD, a low-competition second main pot, with zero rebuild.

Every panel follows the pattern Frank's voice orb already proved: browser panel → /api/<name> relay → LLM + tools → one spoken answer. The data feeds can start mocked for the demo and swap to real IoT / WMS / BIM later.

Demo runs headset-free in the PICO emulator (pnpm --filter frontend dev:spatial) — and the whole floor-walk arc doubles as the trailer shot list, where 25% of the score lives.
