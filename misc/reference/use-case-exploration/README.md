# Use-Case Exploration

Which XRCC use-case track should Looka enter as its *second* pot? Looka is a spatial
room of AI-agent panels, so the same PICO build can enter a Use Case track with **zero
rebuild** — we just change what the panels look at. This folder is that exploration.

**Open [`index.html`](./index.html) first** — it's the hub: a timeline of how the thinking
moved, the cross-cutting insight, and links to both outcomes.

## The takeaway

The industrial doc came first. Then feedback from Dinesh + the insurance team flipped the
frame: insurers don't want the flashy field demo — they want the **back-office paperwork
gone** (contracts, forms, email, the claims queue). That's the least-contested,
highest-opportunity track, and the most on-idea fit for a room of ops agents.

**Open follow-up:** validate with the ERGO people at demo day — six questions are queued in
the insurance doc. Their answers re-rank the ten and lock the four.
